# 聚合结果与解释

## 1. MemConflict100：当前最适合的主要贡献线

历史 strict-contract local main 100-case 结果为 Direct `58/100`、QVF `67/100`，形成 `9 wins / 0 losses`。它直接测试同一用户长期记忆中存在冲突、更新或条件差异时，检索后有效性控制能否减少错误引用，因此与 QVF 的研究问题最接近。

该分数属于受保护的历史实现与协议。2026-07-16 精简代码是后续工程候选，尚未完成等价 full refresh，因此不能把 `67/100` 写成精确当前 working tree 的新分数。

## 2. LongMemEval-S500：普通长期记忆的非退化门槛

历史 official full benchmark 为 Direct `195/500`、QVF `195/500`，QVF losses 为 0。这个结果不表示普通 recall 得到显著提升；它支持的是选择性控制没有破坏原本能够回答的普通长期记忆问题。

## 3. LoCoMo79：探索性信号

adapter-ready 79-case supporting pilot 为 `40/79 -> 43/79`，3 wins / 0 losses。它只是 pilot，不能称为 full LoCoMo；覆盖不足可能来自抽取与 adapter，而不是核心 validity policy。

## 4. STALE400：机制上限而非纯核心增益

Direct full400 新结果为 `177/1200 = 14.75%`，case-cluster 95% CI 为 `[12.58%, 16.92%]`。历史 full QVF 描述为 `1088/1200 = 90.67%`，两者不是同一时间、同一精确代码树的 paired run。

STALE adapter 已提供 400/400 exact old/new pair、同实体同 slot、old-before-new 时间和 old-premise binding，因此 `+75.92pp` 应描述为 **oracle-pair validity mechanism + reader contract 的联合上限**，不能作为 raw retrieval 或 pure-core QVF 的独立归因。

固定 adapter 后，更强的组件证据为：

| 方法 | 正确数 | 准确率 |
|---|---:|---:|
| Full QVF | 1088/1200 | 90.67% |
| Annotation-only | 1054/1200 | 87.83% |
| No-stale-blocking | 1054/1200 | 87.83% |

因此完整 stale/current blocking policy 的可归因净增量为 `34/1200 = 2.83pp`。

## 5. 上游检索与跨模型组合

GPT-4o-mini 的 frozen retrieval paired controls 显示：BM25 `58 -> 62`、dense `58 -> 61`、hybrid `62 -> 65`。这支持 QVF 可以消费不同外部 retriever 的冻结输出。

但跨模型结果方向不一致：Claude Haiku 4.5 在 frozen BM25 上 `73 -> 69`，Gemini 3.1 Flash-Lite 为 `61 -> 61`。因此当前正确结论是“接口可组合，但 accuracy gain 尚未跨模型成立”。

## 6. 当前 GO / NO-GO

- **GO**：QVF 作为 retriever-agnostic post-retrieval validity controller 的接口位置与核心运行闭包。
- **GO**：stale/conflict validity 机制继续作为主线；LongMemEval 继续作为 ordinary-memory safety gate。
- **NO-GO**：普适 accuracy plug-in、跨模型 zero-loss、pure-core STALE +75.92pp、end-to-end token saving。
- **NO-GO**：当前 learned/adaptive Router 晋升。已有 semantic candidate `42/48 = 87.5%`，低于预注册 overall `90%` 门槛。
