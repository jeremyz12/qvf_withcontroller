# 证据边界与已知限制

## 可以支持的结论

- QVF 的位置和接口已经明确：外部检索之后、答案生成之前。
- 核心运行闭包可以独立处理 raw/structured input、provenance、temporal validity、semantic relation、admission、sidecar 和 controller action。
- 历史 MemConflict100 结果支持 conflict/stale validity 的正向贡献线。
- 历史 LongMemEval-S500 结果支持 ordinary-memory 的零损失非退化门槛。
- BM25、dense、hybrid 以及三家 answer provider 的 transport 与 contract 已经跑通。

## 当前不能支持的结论

- 不能声称 exact current tree 已复现所有历史 benchmark 分数。
- 不能把 STALE400 的 `+75.92pp` 全部归给 pure-core QVF。
- 不能声称跨模型普遍提升；Claude 出现负向、Gemini 为净零。
- 不能声称 learned/adaptive Router 已经可部署。
- 不能声称端到端 token saving；当前 validity judgment、抽取或 retrieval repair 可能增加调用。

## 当前工程残余风险

- active Router 仍是 deterministic heuristic baseline；通用线索虽不是领域词表，但跨语言、隐式冲突和长距离语义仍可能漏检。
- response dimension、condition attachment 和 set cardinality 的语义 provider 尚未完成独立自然数据双标注。
- 历史 protected scores 与 2026-07-16 engineering candidate 之间需要一次冻结版本 canary/full refresh 才能重新绑定。
- 发布包只有一个 smoke test，完整研究仓库中的大量评测与回归测试未包含在此精简仓库中。
