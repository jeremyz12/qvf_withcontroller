# 相关工作与下一步

## 建议优先阅读

1. **STALE: Benchmarking the Temporal Robustness of Large Language Models in Dynamic and Evolving Knowledge Environments**：提供专门的 stale premise、temporal robustness 测试，但要区分 oracle old/new pairing 与真实检索。
2. **MemConflict: An Evidence-Based Memory Conflict Benchmark for Long-Term Conversational Agents**：最接近 QVF 的主要问题，强调长期会话中的证据冲突与更新。
3. **LongMemEval: Benchmarking Chat Assistants on Long-Term Interactive Memory**：用于验证 ordinary long-memory recall 是否被控制器破坏。
4. **LoCoMo: Long-term Conversational Memory for Large Language Models**：覆盖长对话事件、关系和时间推理，但当前 QVF 证据只覆盖 79-case pilot。
5. **When Facts Change: Evaluating Language Models' Ability to Update Their Knowledge**：帮助解释新旧事实并存时，模型为何会受到旧知识和旧 premise 锚定。
6. **How Memory Management Impacts LLM Agents**：说明 memory selection、compression 和 management policy 本身会改变 agent 行为，支持把检索后控制视为独立系统层。
7. **TReMu: Temporal Retrieval-Augmented Generation with Adaptive Temporal Ordering**：可作为上游 temporal retrieval 对照；QVF 仍放在其冻结输出之后。
8. **Mem0 / Hindsight / APEX-MEM**：代表不同 memory 写入、组织与检索设计，适合测试 QVF 的 retriever-agnostic composability。
9. **EvolveMem / Self-Harness / UMEM**：为 constraint-preserving policy evolution、failure trace、held-out promotion 和 rollback 提供方法参照；不能直接把在线 benchmark residual 变成规则。

## 下一步三项优先工作

1. 对跨模型 11 个 loss 做方法盲双审，区分 controller harm、judge 偏差、答案表达差异和检索缺失。
2. 建立独立自然 48-case response-dimension 双标注，训练 factorized semantic provider，并使用 calibrated abstention；晋升要求 validity-risk recall、ordinary false-positive、ECE 和 zero false-safe 同时通过。
3. 在冻结 BM25/dense/hybrid 上先报告 pair coverage，再以同一代码版本运行 Direct、always-on QVF 和 selective controller；没有跨模型同向和 ordinary-recall zero-loss 时停止晋升。

## 不采用的捷径

- 不从 STALE、MemConflict、LongMemEval 或 LoCoMo 的问题和 residual wording 归纳触发短语；
- 不在线积累关键词；
- 不使用无条件 recency；
- 不把 adapter/extraction 修复写成 core QVF gain；
- 不把 token 下降作为结论，除非 end-to-end calls 和 tokens 在同等或更高准确率下确实减少。
